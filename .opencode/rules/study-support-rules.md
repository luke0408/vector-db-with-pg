# 스터디 서포트 규칙

이 규칙은 본 프로젝트에서 수행되는 스터디성 개발 작업의 재현성과 학습 기록 품질을 보장하기 위한 필수 규칙이다.

## 적용 대상 (필수)

아래 작업 유형에 해당하면 반드시 결과 문서를 남긴다.

1. 쿼리 튜닝 (인덱스 추가/변경, SQL 변경, 플래너 힌트/설정 변경)
2. 기능 수정/구현 (동작 변경이 발생하는 API, 서비스, UI, 배치 로직)
3. 성능 개선 (지연 시간, 처리량, 리소스 사용량 관련 최적화)

## 문서화 강제 규칙

1. 작업 완료 전, 반드시 `docs/` 하위에 보고서를 신규 작성하거나 기존 보고서를 업데이트한다.
2. 성능 관련 작업은 반드시 **전/후 비교**를 포함한다.
3. 비교 수치가 없으면 작업 완료로 간주하지 않는다.
4. 재현 가능한 명령(실행 커맨드/환경/파라미터)을 반드시 포함한다.
5. 확인 가능한 근거(EXPLAIN ANALYZE, 테스트 결과, 로그 요약)를 반드시 포함한다.

## 성능 작업 문서 최소 필수 항목

1. 목적/범위
2. 측정 환경 (데이터 크기, DB/런타임 정보)
3. 변경 전 결과 (시간, 핵심 플랜 노드)
4. 변경 후 결과 (시간, 핵심 플랜 노드)
5. 비교 표 (개선율 포함)
6. 재현 절차
7. 리스크/트레이드오프 및 다음 튜닝 포인트

## 기능 수정 문서 최소 필수 항목

1. 변경 배경
2. 변경 파일 목록
3. 동작 변경 요약 (사용자 영향)
4. 검증 방법 (테스트/수동 검증)
5. 롤백 또는 안전장치

## 파일 네이밍 가이드

- 권장 경로: `docs/`
- 권장 형식: `<작업영역>-<주제>-<YYYY-MM-DD>.md`
- 예시: `docs/hybrid-search-hnsw-ivfflat-benchmark-2026-03-01.md`

## 품질 게이트

- 문서 누락 시: 작업 미완료로 처리
- 전/후 수치 누락(성능 작업): 작업 미완료로 처리
- 재현 절차 누락: 작업 미완료로 처리
